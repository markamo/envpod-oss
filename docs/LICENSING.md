### What BSL 1.1 Means

1. **You can use envpod freely** — run it, modify it, self-host it, all at no cost
2. **You can read and learn from the source** — full transparency, no obfuscation
3. **Competitors can't embed or resell it** — commercial use that competes with envpod requires a separate license
4. **It becomes open source automatically** — on 2030-03-07, the code converts to AGPL-3.0

### What You Can Do

- **Use envpod internally** at any scale, for any purpose (personal, commercial, government)
- **Modify the source** for your own use
- **Self-host** on your own infrastructure with no restrictions
- **Build tools that interact with envpod** via CLI — BSL applies to the envpod codebase, not to programs that invoke it

### What Requires a Commercial License

- Offering envpod as a hosted/managed service to third parties
- Embedding envpod into a competing commercial product
- Redistributing envpod as part of a competing offering

### Why BSL?

Every AI agent sandbox in the market uses Apache-2.0 or MIT. That means any cloud provider can take the code, add a UI, and sell it as a service — giving nothing back. BSL protects envpod's governance innovations while keeping the software free for every user and team to self-host.
