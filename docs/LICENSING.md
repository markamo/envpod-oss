### Why AGPL?

The AGPL is the strongest copyleft license available. It ensures that:

1. **You can use envpod freely** — run it, modify it, distribute it, all at no cost
2. **Modifications stay open** — if you modify envpod and distribute it (or offer it as a service), you must release your changes under AGPL-3.0
3. **Cloud providers can't embed it** — the AGPL's network clause means that Docker, AWS, or any cloud provider offering envpod as a managed service must open-source their entire stack under AGPL-3.0

This protects the project from being absorbed into proprietary platforms while keeping it fully free for individual developers, teams, and companies running it on their own infrastructure.

### What You Can Do

- **Use envpod internally** at any scale, for any purpose (personal, commercial, government)
- **Modify the source** for your own use
- **Distribute modified versions** as long as you provide source code under AGPL-3.0
- **Build tools that interact with envpod** via CLI — your tools don't need to be AGPL (the license applies to the envpod codebase, not to programs that invoke it)

### What Requires a Commercial License

- Embedding envpod libraries into proprietary software
- Offering envpod as a managed cloud service without open-sourcing your stack
- Removing AGPL obligations for redistribution
