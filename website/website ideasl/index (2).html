FS

Filesystem

Copy-on-write overlay. Agent sees a read-only snapshot. Every write captured in a private layer. Diff, commit individual files, or rollback everything.

NET

Network

Own network namespace with dedicated veth pair. Default deny. Per-pod DNS whitelist with anti-tunneling protection. Raw IP connections blocked.

PID

Process

PID + UTS + user namespace isolation. cgroup v2 hard limits on CPU, memory, I/O, and process count. Seccomp-BPF syscall filtering.

DEV

Devices

Selective device passthrough. GPUs work out of the box — NVIDIA and AMD. No drivers, no toolkit, no configuration.

DSP

Display

Governed Wayland forwarding with per-window isolation. X11 fallback available. Agents run browsers and GUI apps inside governed pods.

SND

Audio

PipeWire or PulseAudio forwarding. Playback only by default — microphone requires explicit opt-in. One toggle in pod.yaml.

Governance

One ceiling

Isolation prevents escape. Governance prevents damage. Every action goes through your approval before it becomes permanent.

Q

Action Queue

Four tiers: immediate, delayed, staged, blocked. The agent submits actions via Unix socket. You decide what runs, what waits, what's denied. Every action is undoable.

CAT

Action Catalog

20 built-in action types — HTTP, filesystem, git, custom. MCP-style tool discovery. Agent can only call what you've listed. Hot-reload without restart.

VLT

Credential Vault

ChaCha20-Poly1305 encrypted at rest. Injected as env vars at runtime — never in config, CLI args, or logs. Import from .env. Bind to specific domains.

SNP

Snapshots

Named checkpoints. Auto-snapshot before every run. Restore to any point. Promote snapshots to bases for instant cloning. Configurable retention.

CLN

Base Pods & Cloning

Setup once, clone instantly. 130ms clone time. Spin up 50 identical agents in under a minute. Each gets its own isolated overlay.

AUD

Audit & Monitoring

Append-only JSONL trail. Static security analysis before deployment. Runtime monitoring with auto-freeze, log, kill, or restrict on alert.

Operations

Built for production

Remote control, live mutation, web dashboard, pod discovery — without restarting anything.

RC

Remote Control

Freeze, resume, kill, or restrict a pod live. Update DNS, ports, and resource limits on a running pod. No restart required.

WEB

Web Dashboard

Fleet overview, live cgroup stats, audit timeline, diff viewer with commit/rollback buttons. No database, no dependencies.

DSC

Pod Discovery

Pods find each other by name. Bilateral — both must opt in. Central DNS daemon with live mutation. Revoke access instantly.

Workflow

How envpod works

Complete agent lifecycle — sandboxed, governed, auditable, reversible.

01

Define the pod

One YAML file: network policy, resource limits, device access, action catalog, security profile. envpod audit --security checks for misconfigurations before you deploy.

02

Agent sees your system through a governed overlay

Host filesystem is the read-only base layer. Agent reads your codebase, runs your tools — all writes go to the overlay. Sub-second start.

03

Every action goes through the queue

Push code, send an email, call an API — each hits the four-tier queue. Immediate actions execute. Staged actions wait for envpod approve. Blocked actions are denied.

04

Credentials are invisible to the agent

API keys live in the encrypted vault. Envpod injects them at execution time. The agent never sees the real secret.

05

You review the diff

envpod diff shows every file changed. Commit individual files with envpod commit path/to/file. Export to staging first with --output.

06

Nothing persists without your approval

envpod rollback discards everything. envpod snapshot restore returns to any checkpoint. envpod destroy — no traces.
