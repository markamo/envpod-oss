Containers isolate processes.

Envpod governs autonomous side effects.

Isolation

Five walls

Every attack surface isolated. The agent runs inside a pod with hard boundaries enforced by the Linux kernel.

FS

Filesystem

Copy-on-write overlay. Agent sees a read-only snapshot. Every write captured in a private layer. Diff, commit individual files, or rollback everything.

NET

Network

Own network namespace with dedicated veth pair. Default deny. Per-pod DNS whitelist with anti-tunneling protection. Raw IP connections blocked.

PID

Process

PID + UTS + user namespace isolation. cgroup v2 hard limits on CPU, memory, I/O, and process count. Seccomp-BPF syscall filtering.

DEV

Devices

Selective device passthrough. GPUs work out of the box — NVIDIA and AMD. No drivers, no toolkit, no configuration. Real hardware through the overlay.

DSP

Display

Governed Wayland forwarding with per-window isolation. X11 fallback available. Agents can run browsers and GUI apps inside governed pods.

SND

Audio

PipeWire or PulseAudio forwarding. Playback only by default — microphone requires explicit opt-in. Auto-detected, one toggle in pod.yaml.

Governance

One ceiling

Isolation prevents escape. Governance prevents damage. Every action goes through your approval before it becomes permanent.

Q

Action Queue

Four tiers: immediate, delayed, staged, blocked. The agent submits actions through a Unix socket. You decide what runs, what waits, and what's denied. Every action is undoable.

CAT

Action Catalog

20 built-in action types — HTTP, filesystem, git, and custom. The agent discovers available tools at runtime (MCP-style). It can only call what you've explicitly listed. Hot-reload without restart.

VLT

Credential Vault

ChaCha20-Poly1305 encrypted at rest. Secrets injected as env vars at runtime — never in config files, CLI args, or logs. Import from .env files. Bind credentials to specific domains.

SNP

Snapshots

Named checkpoints of overlay state. Auto-snapshot before every run. Restore to any point. Promote snapshots to base pods for instant cloning. Configurable retention.

CLN

Base Pods & Cloning

Run setup once, clone instantly. 130ms clone time — 10× faster than re-init. Spin up 50 identical agents in under a minute. Each gets its own isolated overlay.

AUD

Audit & Monitoring

Append-only JSONL audit trail. Static security analysis of pod.yaml before deployment. Runtime monitoring with configurable thresholds — auto-freeze, log, kill, or restrict on alert.

Operations

Built for production

Remote control, live mutation, web dashboard, pod discovery, and fleet management — without restarting anything.

RC

Remote Control

Freeze, resume, kill, or restrict a pod live via Unix socket. Update DNS rules, port forwarding, and resource limits on a running pod. No restart required.

WEB

Web Dashboard

Fleet overview, pod detail with live cgroup stats, audit timeline, diff viewer with commit and rollback buttons. No database, no external dependencies.

DSC

Pod Discovery

Pods find each other by name via agent-b.pods.local. Bilateral — both pods must opt in. Central DNS daemon with live mutation. Revoke access instantly.

Workflow

How envpod works

Complete agent lifecycle — sandboxed, governed, auditable, reversible.

01

Define the pod

One YAML file: network policy, resource limits, device access, action catalog, security profile, audit settings. envpod audit --security checks for misconfigurations before you deploy.

02

Agent sees your system through a governed overlay

Host filesystem is the read-only base layer. Agent reads your codebase, runs your tools, installs packages — all writes go to the overlay. Sub-second start, no images to pull.

03

Every action goes through the queue

Agent requests to push code, send an email, or call an API — each action hits the four-tier queue. Immediate actions execute. Staged actions wait for your envpod approve. Blocked actions are denied.

04

Credentials are invisible to the agent

API keys live in the encrypted vault. When an action needs credentials, envpod injects them at execution time. The agent never sees the real secret — only the result.

05

You review the diff

envpod diff shows every file the agent added, modified, or deleted. Commit individual files with envpod commit path/to/file. Export to a staging directory first with --output.

06

Nothing persists without your approval

envpod rollback discards everything. envpod snapshot restore returns to any checkpoint. envpod destroy removes the pod entirely. No traces on the host.

Architecture

Pluggable isolation backends

Choose your isolation level. The governance ceiling stays the same.

Available now

native

Linux namespaces + cgroups v2 + overlayfs. Fastest start, lowest overhead. GPUs work natively. x86_64 and ARM64 — laptops, servers, Raspberry Pi, NVIDIA Jetson.

Roadmap

gvisor

User-space kernel intercepts all syscalls. No shared kernel risk. Stronger isolation for production agents where kernel exploits are a concern.

Roadmap

firecracker

MicroVM with dedicated kernel. Hardware-level isolation. For regulated industries, defense, and maximum-security deployments.

All backends share the same governance ceiling:
action queue · credential vault · action catalog · audit log · overlay review · DNS filtering · monitoring · remote lockdown · pod discovery
