One YAML file. Five walls. A governance ceiling.

You write pod.yaml. The agent runs with kernel-level isolation on all sides and a governance layer that intercepts every significant action.

🗂️
Copy-on-Write Filesystem
Every agent write goes into an OverlayFS overlay. Your host filesystem is untouched until you explicitly approve. envpod diff shows the exact changeset. envpod commit or envpod rollback in one command. Named snapshots let you restore any point mid-run.
🔐
Encrypted Credential Vault
Secrets encrypted at rest with ChaCha20-Poly1305. Injected as environment variables at runtime only — they never appear in pod.yaml, command lines, shell history, or audit logs. envpod vault import .env to bulk-load from existing files.
📋
Action Queue & Approval
20 built-in action types: HTTP (6), filesystem (7), git (6), custom (1). Every call is validated, tiered, and audited. Four tiers: immediate, delayed, staged (human approval), blocked. Hot-reload — change tiers without restarting.
🌐
DNS Filtering
Each pod runs its own embedded DNS resolver. Define an exact allow-list of domains the agent can reach. Anti-DNS-tunneling protection built in. Live mutation — add or remove domains on a running pod with no restart.
📊
Structured Audit Log
Every lifecycle event recorded in an append-only JSONL file: actions, approvals, vault access, DNS queries — all timestamped. envpod audit --security runs static analysis on your pod.yaml for misconfigurations before you ever run.
🖥️
Web Dashboard
envpod dashboard starts a local web UI on port 9090. Fleet overview with live cgroup stats, filterable audit timeline, diff viewer with commit and rollback buttons. No database, no external dependencies — reads pod state directly.
🔒
Process Isolation
PID namespace, cgroups v2 CPU/memory/IO limits, seccomp-BPF syscall filtering, UTS and user namespaces. A runaway agent cannot starve the host. A compromised agent cannot see host processes or call dangerous syscalls.
⚡
Base Pods & Fast Cloning
Run envpod init once with all your setup commands. That becomes a base. Clone from it in ~130 ms — 10× faster than re-running setup. Spin up a fleet of identical agents in seconds, each with its own isolated overlay.
📡
ARM64 & Embedded
The same static musl binary runs on x86_64 and ARM64 with no runtime dependencies. Tested on Raspberry Pi 4/5 and NVIDIA Jetson Orin with full GPU passthrough via /dev/nvhost-*.

The governance workflow

Four commands cover the entire agent lifecycle — from creation to review to commit.

1
Define the pod
Declare isolation rules, DNS allow-list, resource limits, and vault keys in a single pod.yaml. Run setup commands to install dependencies — snapshot the result as a base.
envpod init my-agent
2
Run the agent
The agent runs inside the pod. All writes go to the overlay. The host filesystem is completely untouched. DNS filtering blocks unexpected network calls.
envpod run my-agent -- claude
3
Review changes
Inspect every file the agent created, modified, or deleted. Nothing is permanent yet. Approve staged actions in the queue. Check the audit log for the full timeline.
envpod diff my-agent
4
Commit or rollback
Apply approved changes to the host filesystem. Or discard everything in one command — the host is exactly as it was before the run.
envpod commit my-agent

Nothing else does both

No other tool combines OS-level isolation with governance. The intersection is where envpod lives.

Docker

Isolates.

Draws a box around the process. Doesn't govern what happens inside it. Container writes are permanent. No action approval. No credential interception. No audit trail.

E2B / Daytona

Governs.

Managed sandboxes in the cloud. Your agent's code runs on their infrastructure. Requires a network connection, an account, and trust in a third party.

Firejail

Sandboxes.

Syscall filtering and filesystem restrictions. No action queue. No credential vault. No overlay-as-review-gate. No way to inspect and approve what the process did.

envpod

Isolates and governs.

OS-level isolation on your own machine. Plus: every write captured in overlay, every action queued and tiered, every secret protected, every decision audited. On-premise. No third party.

Install

Single static binary. No daemon, no runtime dependencies. Requires Linux with cgroups v2.

# One command — detects your architecture, downloads and installs
$ curl -fsSL https://envpod.dev/install.sh | sudo sh
# Raspberry Pi 4/5 (64-bit OS) or NVIDIA Jetson Orin — same command
$ curl -fsSL https://envpod.dev/install.sh | sudo sh
# Requires Rust toolchain — rustup.rs
$ git clone https://github.com/markamo/envpod-oss
$ cd envpod-oss && cargo build --release
$ sudo cp target/release/envpod /usr/local/bin/

Try it in 30 seconds:

$ sudo envpod init hello
$ sudo envpod run hello -- bash -c "echo 'governed' > /home/agent/hello.txt"
$ sudo envpod diff hello
$ sudo envpod commit hello # or: sudo envpod rollback hello
Requirements: Linux kernel 5.15+  ·  cgroups v2  ·  OverlayFS  ·  root (for namespace setup)  ·  ~12 MB disk
