🗂️
Copy-on-Write Filesystem
Every write goes to an OverlayFS overlay. Host untouched until you explicitly approve. envpod diff shows the changeset. envpod commit or envpod rollback in one command. Named snapshots with auto-checkpoint, restore, and promote-to-base.
🔐
Encrypted Credential Vault
ChaCha20-Poly1305 encrypted at rest. Injected as env vars at runtime — never in pod.yaml, CLI args, shell history, or logs. envpod vault import .env to bulk-load. Bind credentials to specific domains.
📋
Action Queue & Approval
20 built-in action types: HTTP (6), filesystem (7), git (6), custom (1). Four tiers: immediate, delayed, staged (human approval), blocked. MCP-style tool discovery. Hot-reload without restart.
🌐
DNS Filtering
Each pod runs its own embedded DNS resolver. Exact allow-list of reachable domains. Anti-DNS-tunneling protection. Live mutation — add or remove domains on a running pod with no restart.
📊
Audit Log & Security Scan
Append-only JSONL. Every action, approval, vault access, DNS query — timestamped. envpod audit --security runs static analysis on pod.yaml before you ever run. Runtime monitoring with auto-freeze on alerts.
🖥️
Web Dashboard
envpod dashboard — local web UI on port 9090. Fleet overview, live cgroup stats, filterable audit timeline, diff viewer with commit/rollback buttons. No database, no dependencies.
🔒
Process Isolation
PID + UTS + user namespace. cgroups v2 CPU/memory/IO/PID limits. Seccomp-BPF syscall filtering. A runaway agent cannot starve the host. A compromised agent cannot see host processes.
⚡
Base Pods & Fast Cloning
Run envpod init once. That becomes a base. Clone from it in ~130ms — 10× faster. Spin up 50 identical agents in seconds, each with its own isolated overlay.
📡
Display, Audio & GPU
Wayland forwarding with per-window isolation. PipeWire/PulseAudio passthrough. noVNC audio streaming via Opus/WebM. NVIDIA and AMD GPUs — no toolkit, no drivers. ARM64: Raspberry Pi, Jetson Orin.
🖥️
Web Display & Desktop
Access a pod's graphical desktop from any browser via noVNC. Audio streaming (PulseAudio + Opus/WebM). File upload from host to pod. Auto-connect, custom branding, auto-restart on crash. Set desktop_env: xfce — open localhost:6080 and you're in.
📦
Host App Auto-Mount
List apps in pod.yaml — envpod resolves binaries, shared libraries, and data dirs via which + ldd, then bind-mounts them read-only. No reinstalling Chrome, Python, or Node inside the pod.
📂
Mount Working Directory
mount_cwd: true or envpod run -w — your project directory is visible inside the pod with COW isolation. Agent reads real files, writes go to the overlay. diff/commit/rollback as usual.
👤
Clone Host User
Your username, shell, dotfiles, and workspace directories — cloned into the pod with COW isolation. The agent works in your real environment, but every change is staged for human review. .ssh, .gnupg, .aws excluded by default.

The governance workflow

Complete agent lifecycle — sandboxed, governed, auditable, reversible.

01

Define the pod

One YAML: network policy, resource limits, device access, action catalog, vault keys. envpod audit --security checks for misconfigurations before you deploy.

02

Agent runs inside the overlay

Host filesystem is read-only base. Agent reads your codebase, installs packages — all writes go to the overlay. Sub-second start.

03

Every action goes through the queue

Push code, send email, call API — each hits the four-tier queue. Immediate actions execute. Staged actions wait for envpod approve. Blocked actions denied.

04

Credentials stay invisible

API keys live in the encrypted vault. Envpod injects them at execution time. The agent never sees the real secret.

05

You review the diff

envpod diff shows every file changed. Commit individual files. Export to staging first with --output.

06

Nothing persists without your approval

envpod rollback discards everything. envpod snapshot restore returns to any checkpoint. Auto-snapshots before every run. Promote any snapshot to a clonable base. envpod destroy — no traces.

Quick Start

Single static binary. No daemon, no runtime dependencies. Requires Linux with cgroups v2.

# Auto-detects architecture — prompts for sudo after download
$ curl -fsSL https://envpod.dev/install.sh | sh
# Raspberry Pi 4/5 (64-bit OS) or NVIDIA Jetson Orin
$ curl -fsSL https://envpod.dev/install.sh | sh
# 1. Install WSL2 (PowerShell as Admin)
$ wsl --install Ubuntu-24.04
# 2. Open Ubuntu terminal, then:
$ curl -fsSL https://envpod.dev/install.sh | sudo bash
# Install OrbStack (lightweight Linux VM for macOS)
$ brew install orbstack
$ orb create ubuntu envpod-vm
# Inside the Linux VM:
$ orb shell envpod-vm
$ curl -fsSL https://envpod.dev/install.sh | sudo bash

Runs natively inside a lightweight Linux VM. Full features. Also works with Lima or UTM.

# Requires Rust toolchain — rustup.rs
$ git clone https://github.com/markamo/envpod-ce
$ cd envpod-ce && cargo build --release
$ sudo cp target/release/envpod /usr/local/bin/

Try it in 30 seconds:

$ sudo envpod init hello
$ sudo envpod run hello -- bash -c "echo 'governed' > /home/agent/hello.txt"
$ sudo envpod diff hello
$ sudo envpod commit hello # or: sudo envpod rollback hello

$ sudo envpod vault openclaw set ANTHROPIC_API_KEY
